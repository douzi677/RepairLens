# RepairLens（AI维修报价分析助手）开发规范

> 本文件用于指导 Claude Code 开发 RepairLens 项目。
> 当 PRD 与本文件发生冲突时，以 PRD 为准；当实现方式存在多种选择时，以本文件为准。

---

# 一、项目定位

RepairLens 是一款帮助普通消费者分析家电维修报价是否合理的 AI Web 应用。

核心价值：

> 付款之前，先知道报价是否合理。

RepairLens 不是：

- AI聊天机器人
- 家电维修百科
- 家电维修教学工具
- 官方报价平台

RepairLens 是：

维修报价分析助手（Decision Support Tool）

帮助用户降低维修报价的信息不对称，而不是替用户做决定。

---

# 二、开发目标

整个项目遵循以下目标（按优先级排序）：

1. 产品体验优先
2. 移动端优先
3. 简单、快速、可信
4. 易维护、易扩展
5. 不做过度设计

如果出现多个实现方案，优先选择最简单、最稳定的方案。

---

# 三、MVP原则

MVP 只实现最核心功能。

当前版本只包含：

✅ 选择家电类型

✅ 输入型号（可跳过）

✅ 选择维修项目（支持多选）

✅ 输入维修总报价

✅ 输出报价分析报告



# 四、AI职责

LLM 只负责：

- 理解用户输入
- 理解维修项目
- 生成解释
- 生成维修建议
- 生成风险提示
- 优化文案

LLM 禁止负责：

- 计算价格
- 推测价格
- 编造价格
- 编造维修数据
- 编造市场行情

所有价格必须来自：

data/prices.json

所有价格计算必须使用程序完成。

不得由 LLM 决定。

---

# 五、报价分析规则

分析报告至少包含：

1. 本次维修内容

2. 零件成本区间

3. 人工费用区间

4. 上门费用区间

5. 合理总价区间

6. 用户报价分析

7. 溢价金额（如有）

8. 溢价比例（如有）

9. 推荐成交价

10. 风险提示

11. 温馨提示

12. 免责声明

不要只输出：

"合理"

或

"不合理"

必须解释原因。

---

# 六、UI设计原则

整体风格：

现代、专业、可信。

参考：

- Apple
- Linear
- Notion
- Stripe

页面保持：

- 大量留白
- 卡片布局
- 圆角
- 柔和阴影
- 白色背景
- 蓝色作为品牌色

避免：

- 花哨渐变
- 炫酷动画
- 复杂背景
- 五颜六色

---

# 七、交互规范

整个产品采用：

一步一步填写（Wizard）

每一步只做一件事。

尽可能减少输入。

优先：

按钮

其次：

下拉框

最后：

输入框

所有按钮需要明显可点击。

移动端单手即可操作。

---



# 八、错误处理

如果没有找到对应报价数据：

不要编造。

提示：

"暂未收录该维修项目的市场价格，请谨慎参考，并建议咨询品牌官方售后。"

如果用户输入异常：

给予友好提示。

不要出现程序报错页面。

---

# 十、开发流程

开发必须遵循以下流程：

1. 先阅读 PRD

2. 明确当前任务

3. 给出实现方案

4. 待确认后再修改代码

不要一次性生成整个项目。

每次只完成一个明确目标。

---

# 十一、产品理念

RepairLens 的目标不是告诉用户：

"你被坑了。"

而是帮助用户理解：

- 为什么这个价格合理或不合理
- 哪些费用构成了报价
- 是否值得维修
- 下一步可以怎么做

所有设计都应该围绕：

帮助用户建立维修报价的判断能力，而不是替用户做决定。
